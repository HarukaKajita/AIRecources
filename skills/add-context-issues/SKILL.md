# スキル起動時のワークフロー
スキル起動時下記のワークフローに従って動作する。
対話内で議論されていた問題が複数ある場合は問題ごとに分けてワークフローを実行する。
1. 対話内で議論されていた問題の内容を示すタイトルを決める。タイトルに日本語は使わず英語にする。
2. /issues/<タイトル>/フォルダ内にissue.mdファイルを作成。
3. issue.mdファイルに問題に関する情報を下記の形式で具体的に詳細に記載。

## マークダウン形式

```markdown
## 概要
重要度：<1~100で記載。100が最重要。>

<数行程度で簡潔に内容を記載。>

## 該当ファイル
<問題の該当ファイルのパスをリストで記載。>

### 関連ファイル
<それ以外の修正において参考になる関連性の強いファイルのパスをリストで記載。>

## 内容
<問題点の内容を詳しく具体的に記載。>

### 不明点
<修正後のゴールを定める為に必要な情報が不足している場合はその旨について記載。複数ある場合は順序付きリストなどを使用して可読性高く記載。>

## ゴール
<修正後にどのような状態になっているのが正しいかを具体的に記載。ゴールが定まっていない場合はその旨を記載。>

## 備考
<上記以外の項目で記載しておくべき事項や、ヒントになる情報があればここに記載。>
```
