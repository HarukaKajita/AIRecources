# スキル起動時のワークフロー
1. ユーザーに指定された資料($ARGUMENTS)の内容を全て確認する。
2. 資料に記載された内容に関係するファイルの内容を確認する。
4. ユーザーに指定された資料($ARGUMENTS)と同じ階層にplan.mdというファイルを作成する。
3. 資料に記載された要求を満たす実装・修正の計画を下記のテンプレートに沿って作成する。

# 計画テンプレート
```markdown
## 概要
<数行:何をするのか、なぜするのか、どうするのかを簡潔に計画を示す。>

## 関連ファイル
<指定された資料のパスをリストで記載する。>

## 経緯
<経緯や方針、具体的な対応内容を記載する。>

## タスクリスト
- [ ] <step 1>
- [ ] <step 2>
- [ ] <step 3>
...
タスクリスト作成時に心がける事
- 具体的なファイル名や関数名モジュール名などを明示する。
- 曖昧性を排除して具体的な手順を示す。

## 備考
<上記以外の項目で記載しておくべき事項や、ヒントになる情報があればここに記載。>
```
